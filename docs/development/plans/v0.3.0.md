Phase 1 — Incremental Indexing (3–4h)

Vấn đề: --incremental flag là no-op. FAISSIndex.build() tạo IndexHNSWFlat — loại index không hỗ trợ remove_ids().
Toàn bộ incremental update path là dead code.

Thay đổi:

┌─────────────────────────────────┬─────────┬─────────────────────────────────────────────────────────────────────┐
│ File │ Dòng │ Hành động │
├─────────────────────────────────┼─────────┼─────────────────────────────────────────────────────────────────────┤
│ vector_index/vector_index.py │ 583-586 │ Thay IndexHNSWFlat → IndexFlatL2 wrapped trong IndexIDMap2. Dùng │
│ │ │ add_with_ids() thay add(). │
├─────────────────────────────────┼─────────┼─────────────────────────────────────────────────────────────────────┤
│ vector_index/vector_index.py │ 686-737 │ Sau faiss.read_index(), nếu index không phải IndexIDMap2 → tự động │
│ │ │ gọi \_ensure_idmap2() (backward compat với v0.2.0 indexes) │
├─────────────────────────────────┼─────────┼─────────────────────────────────────────────────────────────────────┤

Risk: Switching sang IndexFlatL2 (exact search) thay HNSW (approximate). Chậm hơn một chút trên repos lớn nhưng đúng
hơn. Repos < 50k files không đáng kể. Release notes cần ghi: "Run wsctx index --force sau khi upgrade để có full
incremental support."

Phase 4 — TOON Benchmark (1–2h, độc lập)

Vấn đề: TOON formatter có comment "do NOT advertise token savings until measured with tiktoken" — nhưng chưa ai đo.

Thay đổi:

1. Tạo benchmarks/toon_vs_alternatives.py — render 10/50/100 files qua TOON, JSON, YAML, MD, XML, đếm tokens bằng
   tiktoken cl100k_base, in bảng so sánh
2. Sau khi chạy, áp dụng rule:


    - Savings < 10% → add DeprecationWarning trong render(), update config validation
    - Savings ≥ 15% → remove "experimental" label
    - 10–15% → giữ experimental, ghi số thực vào docstring
